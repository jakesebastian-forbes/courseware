  • Not developed using templates
  • Modules : Admin, Instructor, and Students
  • Registration. The Instructor is registered by the Admin. Students register via group code upon filling up of required information.
  • Generates group code for each class created. This is randomly generated and can be generated anytime. Has a definite time.
  • Provides Online tutorial (Take a Tour) to help students learn to use the courseware.
  • Notifications: Instructor and student receive notifications for any updates carried out by both.
  • Asynchronous Communication (Thread Discussion): Provides discussion board (ex. Forum) for a particular topic posted by both the Instructor and student.
  • Synchronous Communication (Real time Chat): Instructor may chat a student who is currently online and even not. Capable for group chat, wherein message is sent to all members of the group.
  • Ranking. When final grades are computed, class ranking shows the achievement of the students.
  • Announcement: Instructor may post announcements regarding the course.
  • Filters for Searching: Will show just the type of information you want to see.
  • Backup and Restore: Provides Database Backup and restoring of previously backed up databases.
  • When browsed via mobile phones/tablets, the system shall provide an option for viewing: desktop view or mobile view.

  • Group Creation: Allows Instructor to set up groups. Groups are basically classes for students to join. Groups can be locked.
  • Content Creation: Maybe in the form of text, images, videos or links from a legitimate site as reference.
  • Online File Exchange: Instructors may grade, comment upon and return papers submitted by students.
  • Assessment Tool: Provides Online Testing. Access to tests can be controlled so students have only one opportunity to take each test. The test may be timed.
  • Special Exams. The Instructor may create exams for a specific student or group of students. This is also timed.
  • Gradebook: Provides a communication system such that students can see what their instructors have recorded in gradebooks.
  • User Tracking: Allows instructor to keep records about student’s use of courseware.
  • Post Assignment: Creates an assignment by attaching just about any type of file. Turned in Assignments tells who has and has not turned in an assignment. Submission of assignment is locked automatically after due.
  • Calendar Activities: Instructor may create calendar of activities to be viewed by students enrolled in the class.
